Réponses du questionnaire :

1- Une servlet est une classe qui fait office de "controller" dans notre architecture MVC.
2- Une JSP est une vue de notre MVC
3- La JSP est constituée de HTML principalement, on y met du JAVA pour dynamiser nos pages, toutefois on évite
d'y mettre de la logique, celle-ci doit être contenue dans la servlet associée.
4- Tomcat permet cette structure servlet/JSP, Apache Tomcat est notre serveur pour faire tourner cette structure
5- Nous devons charger notre WAR, Web ARchive.
